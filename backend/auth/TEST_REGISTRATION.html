✅ Password Requirements

Requirement Rule Example
Length Minimum 8 characters ✓ "MyPassword123"
Uppercase At least one A-Z ✓ "MyPassword" contains 'M'
Lowercase At least one a-z ✓ "MyPassword" contains 'y'
Number At least one 0-9 ✓ "MyPassword123" contains '1'
Valid Passwords Avoid common weak passwords ✗ "password123" is blocked

📊 Test Cases

Test Case Input Expected Result
Valid Registration All fields correct, unique email ✓ User created, user_id returned
Duplicate Email Email already exists ✗ Error: Email already registered
Weak Password Password missing uppercase ✗ Error: Must contain uppercase letter
Password Mismatch Password != Confirm password ✗ Error: Passwords do not match
Invalid Email Invalid email format ✗ Error: Invalid email format
Missing Field First name empty ✗ Error: First name is required

🔗 cURL Examples

Valid Registration

curl -X POST http://localhost/MedLink/backend/auth/register.php \ -d "first_name=John" \ -d "last_name=Doe" \ -d "email=john@example.com" \ -d "password=SecurePass123" \ -d "confirm_password=SecurePass123" \ -d "phone=+1234567890" \ -d "gender=Male"

Expected response:

{"success":true,"message":"Registration successful! Please check your email to verify your account.","errors":[],"data":{"user_id":1,"email":"john@example.com","name":"John Doe"}}

Invalid Email (Duplicate)

curl -X POST http://localhost/MedLink/backend/auth/register.php \ -d "first_name=Jane" \ -d "last_name=Doe" \ -d "email=john@example.com" \ -d "password=SecurePass123" \ -d "confirm_password=SecurePass123"

Expected response:

{"success":false,"message":"Validation failed","errors":{"email":"Email is already registered"},"data":[]}

Weak Password

curl -X POST http://localhost/MedLink/backend/auth/register.php \ -d "first_name=Bob" \ -d "last_name=Smith" \ -d "email=bob@example.com" \ -d "password=password123" \ -d "confirm_password=password123"

Expected response:

{"success":false,"message":"Validation failed","errors":{"password":"Password must contain at least one uppercase letter"},"data":[]}
